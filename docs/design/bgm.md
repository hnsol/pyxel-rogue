# BGM Design

BGM生成、再生タイミング、ゲーム乱数からの分離方針。

## BGM（しろもふさん自動演奏ツール）

タイトルBGMは `rogue.py` の `TITLE_BGM_MMLS` で管理する。ダンジョンBGMは `pyxel_rogue/rogue_bgm.py` がフロア深度と HP / 空腹状態から選び、`pyxel_rogue/rogue_bgm_generator.py` で実行時生成する。生成データは `assets/bgm_data/` に置く。フロア帯ごとに `chord` / `base` 候補から同一プレイ中1つを選び、帯が変わる時は直前帯と同じ組み合わせを避ける。1-4F は `(8,9)`, `(9,9)`, `(11,9)`、5-9F は `(8,10)`, `(9,10)`, `(11,10)`、10-14F は `(9,10)`, `(10,10)`, `(11,11)`、15-19F は `(10,11)`, `(11,11)`, `(12,11)`、20-24F は `(10,11)`, `(12,11)`, `(12,12)`、25F以降は `(10,12)` 固定とする。同じフロア帯内でも、階が変わったら新しい探索BGMを曲頭から再生する。

ダンジョンBGM用の乱数はゲーム本体の `RNG` や Python global `random` から分離する。結果画面（quit / death / winner）は直前の探索BGMの `chord` / `base` / 編成を引き継ぎ、`speed=360` にした派生を曲頭から鳴らす。タイトルBGMはタイトル/難易度選択のカーソル移動中は維持し、難易度・スコアボード・オンラインモードなどを決定した時点で停止する。Settings の BGM 項目はダンジョン/結果BGMだけを対象にし、タイトルBGMは現状維持する。

効果音は `docs/superpowers/specs/2026-06-01-sound-effects-design.md` に従い、初期実装では Pyxel ch3 を使う。ダンジョン/結果BGMは通常 ch0-3 を使うが、効果音再生中は ch3 を停止・再生し直さず、効果音終了後に現在のBGMの ch3 だけを復帰する。
