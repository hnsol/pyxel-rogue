# Proximity Message Log 議論メモ

元資料: `/Users/masatora/Downloads/260508_近接メッセージログ.txt`

## 要旨

近接メッセージログは、下部固定ログを廃止し、ダンジョン表示上に一時メッセージを配置する UI である。現行仕様は `DESIGN.md` の `message toast` 節を正とする。

議論では、次の点が価値として整理された。

- 進行方向にメッセージを置くことで、プレイヤーの次の注視点に情報を先回りさせる
- `3x3` ブロック量子化により、位置の安定性と追従性を両立する
- 新着メッセージ単位で位置をラッチし、読んでいる途中にログが動き続けることを避ける
- 最新行を強く、過去行を段階的にフェードさせ、短期文脈だけを画面上に残す
- 最大7行の一時表示と、別途 Log 画面による見逃し救済を分担する
- 後方ブロックを原則使わず、探索の時間軸に逆らう視線移動を避ける
- 下・左優先は、手元資料や書道の手本のような補助資料配置の感覚に近い

## 先行事例との距離

議論内では、近い事例として次が挙がった。

- Floating Combat Text: キャラクター周辺に数字や短文を出すが、移動方向予測や量子化は主目的ではない
- Cogmind Map Dynamics: マップ上へ情報を載せ、固定ログから視線を戻す思想は近いが、ログ自体の動的再配置ではない
- Speech bubble auto-layout: 重要領域を避ける配置問題として近いが、リアルタイムの探索移動とは前提が異なる

wiki では「世界初」とは書かず、「移動方向予測、3x3量子化、後方除外、ラッチ維持を組み合わせた Rogue 系 UI 手法」として説明する。
